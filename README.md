Vous devez proposer une implémentation pour faciliter le suivi de votre collection de Lego et savoir notamment si une pièce de Lego est utilisable par un de vos contacts, c'est à dire pas rattaché à un modèle Lego. Votre solution doit donc respecter les principes d'une application REST et fournir ainsi uniquement les pièces disponibles en trop aux consommateurs de votre API accessible publiquement. Il vous faut donc 1 service qui permet de gérer les pièces de Lego et un service qui permet de gérer votre collection de modèles. L'échange de données entre les 2 services doit s'appuyer sur une API REST également avec une authentification entre les 2 services.

Votre proposition de solution doit être regroupée sur un dépôt GIT dont vous fournirez le lien ci-dessous accessible en public. A la racine du projet doit se trouver un schéma qui récapitule les échanges de données entre les différents services. Votre projet doit être implémenté exclusivement en Python et doit être géré avec uv. Pensez à ajouter quelques tests, ils seront inclus dans l'évaluation.
